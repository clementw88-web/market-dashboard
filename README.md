# Clement Wong

A self-hosted market dashboard that auto-refreshes daily via GitHub Actions and serves on GitHub Pages — no server, no API keys, no cost.

---

## What it does

| Feature | Detail |
|---|---|
| **Data source** | Yahoo Finance via `yfinance` (free, no API key) |
| **Refresh schedule** | Daily at 08:00 SGT (00:00 UTC) |
| **Hosting** | GitHub Pages (free) |
| **Sections** | Macro · Equities · Breadth/Sentiment · Position Calculator |

---

## One-time setup (5 minutes)

### 1 — Create a new GitHub repository

1. Go to [github.com/new](https://github.com/new)
2. Name it `market-dashboard` (or anything you like)
3. Set it to **Public** (required for free GitHub Pages)
4. **Do not** initialise with README — you'll push files manually

### 2 — Push this project to your new repo

```bash
cd market-dashboard          # this folder
git init
git add .
git commit -m "init: market command centre"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/market-dashboard.git
git push -u origin main
```

### 3 — Enable GitHub Pages

1. In your repo go to **Settings → Pages**
2. Under **Source** select **GitHub Actions**
3. Save

### 4 — Run the workflow for the first time

1. Go to **Actions** tab in your repo
2. Click **"Refresh Dashboard Data"**
3. Click **"Run workflow"** → **Run workflow**
4. Wait ~2 minutes for it to complete
5. Your dashboard is live at `https://YOUR_USERNAME.github.io/market-dashboard/`

> After the first run, the workflow runs automatically every day at 08:00 SGT.

## Local development

pip install -r requirements.txt

# Fetch live data (writes data/snapshot.json)
python scripts/fetch_data.py

# Preview in browser

## Customising instruments

Edit `scripts/fetch_data.py` — the `GROUPS` list at the top defines every instrument. Each entry is:

```python
{"ticker": "AAPL", "label": "AAPL · Apple Inc.", "holdings": "optional text"},
```

Tickers must be valid Yahoo Finance symbols. After editing, push to `main` and re-run the workflow.

## How the pipeline works

```
[GitHub Actions — cron 00:00 UTC]
        │
        ▼
  fetch_data.py
  yfinance.download(all tickers, period=1y)
        │
        ▼
  data/snapshot.json  ←── committed back to repo
        │
        ▼
  index.html loads snapshot.json via fetch()
  and renders all tables, breadth cards,
  ticker tape, and trend arrows
```

## Troubleshooting

| Problem | Fix |
|---|---|
| "Could not load snapshot.json" | Run the workflow manually first (Step 4 above) |
| Workflow fails on `yfinance` | Check Actions log; Yahoo Finance occasionally rate-limits — re-run |
| Pages not updating | Check Settings → Pages source is set to "GitHub Actions" |
| Tickers show `—` | That symbol may be delisted or temporarily unavailable on Yahoo Finance |
